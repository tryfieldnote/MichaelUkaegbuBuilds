Issue 01 — CPL and ROII1
Finding
Blended CPL is £683 and cost per CRM lead is £6,886 against £48,200 total spend YTD with zero opportunities created in 2026.
Root cause
Event selection weighted toward large external conferences where the company is one of many exhibitors — low attendee-to-lead conversion, poor lead quality. Existing contact database underutilised — money spent acquiring new contacts at high cost when warm contacts already exist.
Immediate action
Pause or minimise new external conference spend. No new external event commitments above £5k without event value assessment sign-off.
Strategic shift
Shift event mix toward owned events and webinars targeting existing warm contacts. Audit and segment existing database by vertical, seniority, and engagement. Build Q3 owned event calendar before committing further budget.

Strategic rationale

The segmentation framework was built to solve Issue I1 directly — specifically the finding that the existing warm contact database was being ignored while budget was spent acquiring new contacts at £6,886 per CRM-linked lead. By scoring and tiering the existing database across five dimensions, the framework enables owned events and webinars to be targeted at Priority and Warm contacts without any new acquisition spend. The data completeness dimension feeds directly into the CRM hygiene workstream — contacts with incomplete records are flagged for a hygiene pass before any outreach, so the segmentation and data quality improvements compound each other.

What was actually wrong

Wrong event mix

Over 80% of events were large external conferences where the company was one of hundreds of exhibitors. Low conversion, high cost. The existing warm database was being ignored as an audience while money went on acquiring new contacts at 6,886 per verified lead.

No handover process between marketing and sales

No defined roles post-event. Marketing and sales had different views on what a qualified lead looked like, so valid leads were filed away and never followed up. One event had 20 attendees and zero follow-ups.

FieldNote: I built an app to fix the post-event lead black hole

Personal project, Lovable + Claude code + Supabase, deployed on Vercel, 2024 to 2025

Live

Running events for years, I kept seeing the same thing. You have 25 great conversations at a conference, get back Monday, and half those leads are still sitting in a notes app two weeks later. Badge scanners only work if the organiser sets them up. CRM apps are too heavy to open mid-conversation. Most people just use their phone notes app and hope for the best. I built FieldNote to fix that.

Started in Lovable, then extended with real React code written through Claude. Not a drag-and-drop prototype: a production app with five working screens, AI pipeline summaries, voice note capture, and a Kanban pipeline view, deployed on Vercel.

The three problems it solves

Capturing a lead takes too long

Different people use different methods and data ends up scattered everywhere. FieldNote gets a lead captured in under 30 seconds at a noisy event.

Nobody knows who to follow up with first

After a busy event, deciding who to prioritise is a job nobody does well. FieldNote scores automatically from seniority, company fit, and a quick gut-feel tap at the point of capture.

CRM entry gets delayed or skipped

Contacts sit in exports for days. FieldNote syncs straight to HubSpot and creates follow-up tasks by score: Hot gets tomorrow, Warm gets 3 days, Cold gets a week.

Try it live

FieldNote Live on Vercel. Core screens working. HubSpot sync and enrichment coming next.

How it was built

Scaffolded in Lovable

Used Lovable to scaffold the React and TypeScript frontend and wire it up to Supabase quickly. Core navigation, capture form, leads list, and pipeline view.

Extended with real code through Claude

The Lovable scaffold was extended with actual React component code written through Claude: AI pipeline summary, voice note capture, per-card action suggestions, score-based routing, and the overdue flagging system.

Supabase for data

Handles database, auth, and real-time data across sessions. Foundation for team mode later.

Deployed on Vercel

Live production deployment, not a local prototype. Any device, shareable URL, continuous deployment.
